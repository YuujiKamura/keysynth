# Brief: Tier 1.4 — Soundboard mode density 16 → 32+

Reference: Issue #32 in YuujiKamura/keysynth. Tier 1.4 of the
Pianoteq-beat campaign.

## Working tree

- Worktree: `<tmp>/ks-soundboard` (e.g. `%TEMP%\ks-soundboard` on
  Windows, `/tmp/ks-soundboard` on Unix)
- Branch: `claude/piano-t1-soundboard-density` (already checked
  out, based on `origin/main`)
- DO NOT touch any file outside this worktree.

## Goal

Raise the per-note partial count cap in the modal LUT from 8 to 32
so `Engine::PianoModal` projects denser harmonic content. The
existing 16-mode soundboard in `Engine::Piano` is hand-picked; the
LUT used by `Engine::PianoModal` caps partials at 8 in the
extractor today. Real piano spectra have 30+ audible partials per
note. Bumping the cap closes a measurable gap against the SFZ
Salamander reference.

## Implementation

1. **Locate the partial extractor.** It's in
   `src/bin/build_modal_lut.rs` or `src/extract/`. Search for the
   constant `8` near a comment about "partials" / "modes". Don't
   change the LUT shape unless you can't avoid it — modify only
   the cap.

2. **Bump cap to 32.** Add a `--max-partials N` CLI flag to
   `build_modal_lut` so future changes don't require recompile.
   Default 32. Document in the bin's help text.

3. **Re-extract LUT JSON.** Run:
   ```
   cargo run --release --bin build_modal_lut -- \
     --reference-dir bench-out/REF/sfz_salamander_multi/ \
     --max-partials 32 \
     --out bench-out/REF/sfz_salamander_multi/modal_lut.json
   ```
   Commit the regenerated `modal_lut.json` (it's already tracked
   per `.gitignore` exception).

4. **Verify in `voices/piano_modal.rs`.** The voice's `from_lut`
   path must handle 32 partials per note without performance
   regression. Check the resonator allocation site — if it's
   bounded by a fixed array size, raise the bound or convert to
   `Vec`.

5. **Verify in the wasm build.** The LUT JSON is `include_bytes!`-
   embedded into the worklet wasm; bigger LUT = bigger wasm. Check
   the size delta. Acceptable if < 200 KB total bump (today's LUT
   is ~17 KB, expect ~50–80 KB at 32 partials).

6. **A/B WAV renders.** Use the existing `render_full_bus` binary:
   ```
   cargo run --release --bin render_full_bus -- \
     --engine piano-modal --notes 60,64,67 --sr 48000 \
     --duration 4 --hold 2 --out bench-out/piano-soundboard/before.wav
   ```
   Render `before.wav` from `origin/main` (the 8-partial baseline)
   and `after.wav` from this branch. Commit both.

7. **MR-STFT residual L2.** Run `score` against the SFZ Salamander
   chord ground truth on `before.wav` and `after.wav`. Expected:
   the 32-partial render reduces residual L2 by ≥ 3 dB on the
   C4 chord test. Report the actual numbers in the PR description.

## Hard rules

- **No silent fallbacks.** If extraction at 32 partials produces
  fewer than expected for a note (e.g. spectral peak picker can't
  find 32 stable peaks), log it loudly per-note. Don't pad with
  zeros.
- **`cargo fmt --check` clean.**
- **Native build clean:** `cargo check --bin keysynth`.
- **Web build clean:** `cargo check --no-default-features
  --features web --target wasm32-unknown-unknown --bin
  keysynth-web`.
- **DO NOT touch:**
  - `src/voices/piano.rs` — T1.1 (Stulov) territory
  - `src/voices/string_inharmonicity.rs` — T1.3 (Fletcher) (if it
    exists from another agent)
  - `src/voices/hammer_stulov.rs` — T1.1
  - `src/voices/longitudinal.rs` — T1.2 (if it exists)
  - `src/audio_kira.rs` / `src/bin/web.rs` / `Cargo.toml`
  - You touch ONLY `src/bin/build_modal_lut.rs`,
    `src/extract/*.rs`, `src/voices/piano_modal.rs` (allocation
    bound only, not DSP changes), and the regenerated LUT JSON.
- **DO NOT change `Engine` / `make_voice` / public API.**
